1 - WHAT IS MIM?
This program was designed to allow 2 or more computer's with network cards or modems (using TCP-IP), to chat with each other by sending text messages in a chatroom. MIM also allows sending files and many more features. MIM is intended for more advanced users and has command line capabilities. If you know little or nothing about networking and the internet, you probably won't be able to use the more advanced features of MIM; however, I will do my best to explain them in this readme file.

2 - REQUIREMENTS
This program needs to have TCP-IP running in order to connect computers together. If a network card is installed, TCP-IP is probably already working.

*If using a Modem, you may need to connect to the internet before MIM can be used.
*If using a network card, it must have TCP-IP running.
*An IP address is required.
*An ISP (Internet Service Provider) is needed for internet computers using DHCP.

*System:

O/S: Windows 95/98/2000/Me/XP/Vista/7.
Minimum Resolution: 640x480

3 - INSTALLATION

MIM does not require any installation, but it may require the Microsoft 2008 Developer kit to be installed. The first time MIM is opened it will create new files to save your settings. Place a shortcut to MIM.exe in your Start/Program Files/Startup folder to have MIM run when your computer starts.

4 - SETTING UP A CONNECTION

A computer can do one of two different jobs. It can be the SERVER (The one that creates the chat room and waits for others to join in) or it can be a CLIENT (the one that joins a chat room that has already been created by a server).

By default, simply press Connect to attempt to join MasterHD's chatroom (which might not be running, but I try to keep the IP updated at all times).

You may also choose to view the server list and choose a server to join from there.

Here is a step-by-step tutorial to set up a custom connection:

CLIENT
1. The First step is to select which job your computer will do, Server or Client. This is done by clicking the "Connection Settings..." button at the top of the window. Here you can select your screen name, port, address, sounds, etc.

2. The Second step is to enter the IP Address of the Server (See Section 6 on how to save these addresses). An IP Address consists of 4 numbers from 0-255 separated by a period (EX: 192.168.0.1) or it can be a DNS assigned name. To find out what your computer's IP address is, see section 7. The Server does not need to enter an IP address.

3. The third step is to enter the Port Number. This can be any integer from 1 to about 20000 (The default is 4001 because no other programs should be using this port). This number specifies which port the application will listen on, like a frequency on a radio. A conflict occurs if another application is using the same port number.

NOTE: Always be sure to enter the Port Number of the server you are connecting to.

4. There are many settings here that you can choose from. They are described in detail later.
SERVER
1. Press the "Server Options" button. Here you can adjust the settings for a chat room that you create (Server).

2. Chatroom Name and Admin Name are the names of the room and admin. They can both be up to 29 characters long.

3. The server can also set the maximum number of clients allowed on the server (1 - 20).

4. The Port number is the port which the server will be listening on for others to join.

5. Enter a password if you want your room to be protected from anonymous users.

6. The last field is the "Screen Users" box. Checking this will notify the admin every time a new user wants to join. The admin has the option of accepting or rejecting them.

7. You will want to know your IP address so that you can tell others and they can join your server. The IP should be displayed in this window. If you are using a router, see Section 9.

8. Lastly, you want to put your "MIM v#.##.zip" (Where #.## is the your current version) file in the same folder as MIM, so that clients connecting with different versions can download yours directly from you.


Press OK. Now select File/Save Settings. You are ready to begin chatting. To create a chat room select Server as the type and press Listen. Everyone else should select Client and press Connect to join a server.

6 - IP ADDRESSES

Saving IP Addresses
A helpful feature that this program offers is the ability to save IP Addresses. Press the "Connection Settings..." button. This will open up a new Window listing all of the saved IP Addresses. select an address to set it as the one that will be used. All addresses will be saved to the file named 'MIM.dat' in the same folder as the program. The address "loopback" and MasterHD's IP address are placed in the list by default (see Troubleshooting section 11.1 for more info on the loopback address). Remember that a server which is NOT using a static IP address may not have the same IP address for more than 24 hours.

NOTE: To save the changes that you have made to any settings, select File/Save Settings or File/Exit. Pressing the close button (the "X" at the top right corner) will not save any changes.

Finding your IP address

Your computer's IP address can be found simply by pressing the Server Options button. Note that if you use a router, this is your local IP, not the one seen by other users outside of the router. You will need to access your router to find the IP, see Section 9.

If that does not work, you can try other methods:
A. Press Start, Run, and type "winipcfg", press OK. Look for IP address.

B. Start the DOS command prompt and type "ipconfig". Look for the address.

C. Right-Click the network connection in the taskbar and select "status". Click the "Support" tab to see the IP address.

7 - SAVING MESSAGES / HISTORY

By default MIM will auto save your conversation messages to the file History0.htm. The /autosave command lets you toggle the autosave setting. You can choose to save the history as HTML or TEXT. The text format will take less hard disk space, but will not save any of the text formatting (color, font, size, etc) from the conversation. You can manually save the conversation by pressing the Save Messages button at the top of the window. You can choose to create a new html/text file, or you can select an existing one. If you select an existing file, the contents will NOT be overwritten; instead, all new conversations will be added to the end of the existing file.

8 - USER LIST FUNCTIONS

The User List Box located to the right of the Message Box displays all of the users currently connected to the server. This allows the Server Admin to manually remove users from the chat room using the "Kick" or "Ban" button. The "Secure Message" button allows the admin to send just 1 user a message box.

The Client has the option of ignoring others by selecting the user and pressing "Ignore User". By pressing it again that user will be un-ignored. The icon next to their names shows whether they are being ignored or not.

  • You can also set your status (away or available)

  • View Profile: View the profile info of a user.

  • Ignore: Ignored any text messages from this user.

  • Send File: Sends files to the user. Note: You can also drag-and-drop files to this window, be sure to select the user first.

  • Send Clipboard: This will send the contents of your Windows clipboard (the ctrl-C and ctrl-V thing) to the specified user! They must have the clipboard format available for it to be of use to them.

My Status (Client only):

  • Available: Normal active status
  • Away: Use this to inform others that you are not at the computer (afk). This can also be automatically set when idle, see connection settings.
  • Disconnect: This will disconnect you from the server.

Admin ONLY:

  • Message: Send a message box with the entered text to the specified user.
  • Kick: Kick the specified user from the server.
  • Ban: Ban the specified user's IP address from the server. User will not be able to rejoin until admin uses the UNBAN command or edits the ban listing in the Server Options window.

9 - Connection Problems

Routers and firewalls

NOTE: Your router and/or firewalls will not prevent you from connecting to a server, however, they could prevent other users from connecting to you.

If you are using a router, you will need to find the router's IP address so that other users outside of the router can connect. You will need to refer to the router's documentation for details. Your ISP will automatically assign your router an IP address just as it would a computer connected directly to the modem. The main idea is to use Port Forwarding so that any incoming requests to the router are forwarded to your computer instead of just being blocked. In most cases, you can access your router's settings at http://192.168.1.1/ From there you must forward the necessary ports (4000-4006 by MIM's default) in order to act as a server and/or send/receive files. If the router asks for admin name and password, refer to the router's documentation for the default name and password if you have not already specified your own.

When sending files, only 1 person has to be able to listen for calls and accept connections. Type /listenable 0 into the console to tell MIM that your computer is behind a router or blocked in some way. If both users are blocked, then you cannot send files to each other. If you know your computer is reachable from the outside internet, be sure to set  /listenable 1 to ensure the most reliable connections, especially when sending files.

10 - Quick Reference

NOTE: These windows cannot be accessed while a connection is open. Most of the values can be adjusted from the console, see the section about Console commands.

Here is quick explanation of each setting:
Connection Settings:
1. Server/Client - Choose whether you are Server or Client.
2. Server Port - Port number of server that you are connecting to.
3. Screen Name - Your name in the room.
4. Allow Server to run commands - Allow server to enter text into your text field
5. Autosave message history - All typed messages are saved automatically to History0.txt in the MIM folder (Section 7).
6. Able to accept calls - Disable this box if you are behind a router or your firewall settings prevent clients from connecting to your computer (for sending files).
7. Auto Login - When MIM starts, it will automatically connect to the port and IP address currently selected. Useful if you also place MIM in the Start/Program Files/Startup folder.
8. IP Address - IP Address of server you are connecting to.
9. Play Sound - Sound that is played whenever a message is received.
10. Image - Image to be displayed in your profile.
11. Internet Browser - You can select which browser will open hyperlinks in MIM.
Server Options:
1. Chatroom Name - Name of chatroom (29 characters).
2. Admin Name - Name of administrator (29 characters).
3. Max Users - Maximum number of users allowed to connect. 
4. Listen Port - port which the server is listening on.
5. Password - Password required in order to join server.
6. Mod Password - Password required for clients to become moderators of the room.
7. Screen Users - Ask admin each time, if a new user can join (Not recommended, use a password or bouncer list instead).
8. Time Stamps - Shows the time a messages was received next to the user name (this is your computer system time and may differ for other users).
9. Public Server - This server will appear in any master server list.
10. Censor Words - Blocks out bad words with asterisks (****)
11. Opening Text - All text in this file (server.txt) will be sent to clients upon joining your chatroom.
12. Bouncer - The bouncer will only let in predefined users. Edit the bouncer.txt file with a username and password for each user.
13. Banned IP's - List of IP's that will not be allowed to join your chatroom.

11 - TIPS
Here are a few easy tips for new users:
1. Make sure that the IP address is correct if you are the client. (see section 3 Setting up a connection, for more details on the IP address).
2. The port address entered by the client must be the same as the port that the Server is listening on.
3. Make sure that the modem or connection device has TCP-IP or is connected to the internet.
4. The server must be reachable, i.e. no firewall or router blocking incoming connections. See Section 9 for details.

12 - TROUBLESHOOTING

If others are having problems connecting to your server or receiving/sending files, it may be a firewall or router problem. See section 9.

1. An easy way to test this program or find a problem, is to simulate a connection between two computers, but by only using a single computer. This can be done by opening the Chatroom program twice so that two instances of the application are running at the same time. Then set one instance as the server and the other instance as the client. Make sure that both of them have the same Port number (EX: 4000) and use "loopback" as the IP address. Next click "Listen" for the server and "Connect" for the client. You should now be able to send messages back and forth between the two applications on just one computer.

2. If the Server cannot "listen" for others then:

*The socket is in use by another application, try another number.
*The device has not been configured in Windows yet.
*No connection device exists.
*A malfunctioning connection device exists.
*TCP/IP is not installed.

3. It should only take a few seconds for the client to connect to the server. If it takes too long an error message will be displayed or the connection will timeout.

Possible causes:
*Wrong IP address,
*Wrong Port number,
*Server is not listening for clients.

4. If the "Port may already be in use" error occurs, change the port number and try again.

14 - Advanced Features (Console Commands)
 The text field is mainly used for typing messages and sending them in the chatroom, but it also functions as a command console. The console offers many commands. Type  /commands to see an alphabetical listing of them, then type /help [command] to see what a specific command does. 

Press the ~ key to cycle through the last 20 previous messages entered.

Available command parameters:
[ID] An integer from 1-20 representing the UserID of the client. Use 0 for admin.
[BOOL] A 0 or a 1 to specify OFF or ON of a Boolean variable.
 [INT]   An integer from 0-32768.
"text" It's a good habit to put quotes around a text parameter if it has spaces in it. Most commands that require only 1 string (/me, /name, etc) don't require quotes, each word will be combined into 1 string automatically. But commands like vote, will need quotes to work properly.

Also note that the command names themselves are NOT case sensitive. They are all converted to uppercase.

There are commands, and then there are variables. The commands usually perform an action, whereas variables just set values.
Here are some common ones:

Commands

    /ping [ID]         Pings a user through the server (requires 1 parameter).
    /me "text"        Places the text after your screen name within asterisks (emote).
    /exit                  Exits the program (Requires no parameters)
    /connect         Connects to the server using the current settings.
    /close              Closes connection to the server/client(s).
    etc.  (To learn what a command does, use the /help command).

Variables

Just enter in the value or string after the command to change its value. See Quick Reference for details on some important variables.

NOTE: Some variables can only be changed while disconnected!

    /bouncer [BOOL]       Sets the server bouncer status ON or OFF (1 or 0).
    /ip "text"                      Sets the current IP address to connect to.
    /name "text"               Sets the current screen name.
    /port [INT]                    Sets the current port to connect to.
    /listenable [BOOL]    Specifies whether users can connect to your computer.
    /playsound [BOOL]  Sets sound ON or OFF (1 or 0).

Moderator (remote admin)

This will allow a client to run admin commands on the server. The server can set the modpass variable with the console or Server Options window. Client will type /mod [password] to request mod status. If password is correct client can type:
    /USER 0 [command]
    The [command] string will be entered and run on the server as if the admin had run the command himself.

Ex.
/USER 0 /KICK 2
    A Moderator would type this to Kick user 2 from the server.

Users (Only works for admin or mod's)

    /kick [ID]
        Kicks the user specified.

    /ban [ID]
        Bans the user's IP address (based on ID number) and kicks the user from the room.

    /addban [address]
        Manually adds an IP address to the banned list.

    /clearban
        Clears the entire banned user's list.

    /censor [0-3]
        Set's the censoring of the server.

    /vote "question" "ans 1 text" "ans 2 text" "ans 3 text" "ans 4 text"
        Displays a vote for all clients. Yes and No are the default answers if none are specified.
        Clients type: /vote [#] to choose one.

    /timestamp [BOOL]
        Display time stamps ON or OFF.

    /tally
        Tallies all the votes even if not everyone has voted yet.

    /user [ID] [command]
        Enters command into the clients console and runs it.
        This example will let a mod run a vote on the server:
        /user 0 /vote "This will work?" yes no "I dont know"

Sending Files

MIM will save the paths of the files into a list and you can use the console to manage them. Dragging and
dropping files will alter (overwrite) the saved list as well.
/addfile [filename]
    Add a file to the list.

/sendfile [ID]
    Send all files in the list to the specified user.

/clearfile
    Clears the list.

/listfile
    Shows all files currently in the list.

Custom Commands

MIM allows you to create custom commands or shortcuts. They are saved in the commands.txt file.
/new [command] [params]

The new command allows you to combine pre-existing commands from MIM into a new custom command.
The best way to explain this is with an example. Lets say you like a website, and would like to create a simple command to take you there:
    /new CAD /followurl http://www.ctrlaltdel-online.com/comic.php

Now simply type (NOT case sensitive):
    /cad

to run the custom command. That example will use the followurl command and open the CAD website.

Even more complex is the use of parameters in the commands. Place a %s somewhere in the command to insert parameters there.
    /new NOTEPAD /run notepad.exe %s

The %s is replaced with the next parameter typed after the custom command name. So by next typing this:
    /notepad Quotes.txt

MIM will actually run this:
    /run notepad.exe Quotes.txt

The first parameter in this example is "Quotes.txt" and it replaces the %s in the custom command.

One more example:

Create: 
    /new SLAP /me slaps %s in the face %s times.

Type into MIM:
    /slap Chris 5

What MIM runs:
    /me slaps Chris in the face 5 times

Output:
    *MasterHD slaps Chris in the face 5 times*

    Type /help [new_command] to see the syntax for the custom command straight from the commands.txt file.
